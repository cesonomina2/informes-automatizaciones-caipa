Extracción de Métricas GA4 - Intranet Casalimpia
Cliente Interno:Casalimpia
Plataforma:n8n (Workflow Automation)
Carpeta N8N:INTRANET CASALIMPIA
Nombre del Flujo:TRAE DATOS INTRANET
Integraciones:Google Analytics 4 (API) & Supabase (PostgreSQL API)
Estado:Producción

1. Resumen Ejecutivo

La automatización "TRAE DATOS INTRANET" es una tubería de datos (Data Pipeline) diseñada para medir la adopción y eficacia de la Intranet Corporativa de Casalimpia. Diariamente, el sistema se conecta a la API de Google Analytics 4 (GA4) para extraer métricas clave de comportamiento (usuarios, interacciones, lecturas completas y búsquedas). Estos datos son parseados, limpiados y enviados a Supabase, creando un repositorio histórico en tiempo real que alimenta el Dashboard Ejecutivo, eliminando por completo la dependencia de reportes manuales o archivos de Excel.

2. Arquitectura de la Solución

2.1. Componentes Tecnológicos

  • Orquestador: n8n (Ejecución programada, transformación JSON y peticiones HTTP).
  • Fuente de Datos (Source): Google Analytics 4 (Autenticación vía OAuth2, Property ID: 522026488).
  • Data Warehouse (Destino): Supabase (PostgreSQL vía API REST).

2.2. Estructura de Base de Datos (Supabase)

La información extraída se distribuye y normaliza en 3 tablas especializadas:

  • 1. 4-intranet_salud_general: Almacena el volumen de tráfico y retención diaria (Métricas globales).
  • 2. 4-intranet_contenidos: Almacena el consumo de las páginas y su profundidad de lectura.
  • 3. 4-intranet_eventos: Diccionario de interacciones del usuario (clics en enlaces de salida, búsquedas en la lupa y scroll depth).

3. Descripción Detallada del Flujo en n8n

El flujo se ejecuta de forma lineal y automatizada mediante tres ramas paralelas que procesan diferentes dimensiones de la analítica.

3.1. Disparador (Trigger) y Extracción Paralela

  1. Schedule Trigger: Orquesta la ejecución a la medianoche, solicitando los datos del día inmediatamente anterior (Yesterday).
  2. Nodo 1: Salud General y Retención (GA4): Extrae `activeUsers`, `engagementRate`, `averageSessionDuration` y `screenPageViewsPerUser` sin dimensiones divididas.
  3. Nodo 2: Contenidos (GA4): Extrae `screenPageViews` divididas por la dimensión `pageTitle` y `pagePath`.
  4. Nodo 3: Eventos e Interacciones (GA4): Extrae `eventCount` dividido por `eventName`, `searchTerm`, `linkUrl` y `pageTitle`.

3.2. Formateo y Limpieza de Datos (Nodos Code)

Los datos crudos de GA4 se inyectan a lógicas de JavaScript para parsear tipos de datos y evitar errores en la base de datos:

  • Formatear Salud: Convierte cadenas de texto en números reales (parseInt y parseFloat) para las tasas de interacción y promedios de tiempo.
  • Formatear Contenidos: Asigna valores null como fallback cuando GA4 no retorna la URL del path (data.pagePath || null).
  • Filtrar Eventos: Aplica una limpieza estricta (If-Statement) dejando pasar a la base de datos únicamente tres tipos de eventos vitales: view_search_results, click y scroll. Además hereda dinámicamente la URL del enlace saliente o, en su defecto, el título de la página navegada.

3.3. Inserción Segura (Upsert a Supabase)

Los tres nodos HTTP Request realizan peticiones POST directas a la API REST de Supabase. Para garantizar la pureza de los datos y prevenir registros duplicados en caso de re-ejecuciones manuales, cada nodo utiliza el parámetro resolution=merge-duplicates con claves únicas (Constraints) específicas:

  • Salud: ?on_conflict=fecha
  • Contenidos: ?on_conflict=fecha,titulo_pagina,ruta_pagina
  • Eventos: ?on_conflict=fecha,nombre_evento,termino_busqueda,url_enlace

4. Escenarios de Transformación de Datos

Escenario 1: Parseo de Tasa de Interacción (Salud)

Entrada cruda de GA4: "engagementRate": "0.795" (String) Transformación en Nodo Code: parseFloat(data.engagementRate) || 0 Salida a Supabase: 0.795 (Float / Numeric real listo para ser calculado en el Dashboard como 79.5%)
Escenario 2: Limpieza de Eventos "Basura"

Eventos obtenidos de GA4: session_start, page_view, scroll, click, first_visit Transformación en Nodo "Filtrar Eventos": if (nombreEvento === 'view_search_results' || nombreEvento === 'click' || nombreEvento === 'scroll') { ... } Salida a Supabase: Solo persisten los eventos de interés (Scroll, Click, Búsquedas). Se reduce el tamaño de la base de datos un 80%.

5. Anexos: Flujo Visual

6. Visualización Final (Dashboard)

Los datos procesados por este flujo de n8n nutren de forma directa y automática el Dashboard Directivo de la intranet. Las métricas calculadas gracias a esta automatización permiten observar los 9 puntos críticos solicitados (Usuarios únicos, fidelidad, tiempo medio, ranking de páginas, eventos de scroll al 90%, clics de salida y búsquedas activas).
